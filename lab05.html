Lab 05 · AI-Assisted Development

Vibe Coding

Introduction

[ Describe what "vibe coding" means — using AI tools to help write, generate, or assist with code rather than writing everything manually. What was the goal of this lab? What were you asked to build or experiment with? ]

What I Built

[ Describe the project you created. What was it — a web page, a script, a tool, an interactive element? What problem did it solve or what did it demonstrate? ]

AI Tools Used

[ Which AI tools did you use to help write the code? (e.g. ChatGPT, Claude, GitHub Copilot, etc.) How did you use them — did you describe what you wanted and let it generate code, or did you ask it to fix/improve code you wrote yourself? ]

Process & Prompting

How I Described What I Wanted

[ What prompts or instructions did you give the AI? Walk through the process — what did you ask first, what did you refine, what worked and what didn't? ]

Iteration & Refinement

[ Did you have to go back and forth with the AI to get the result you wanted? What did you change or improve across multiple attempts? ]

What You Added Yourself

[ What parts did you write, edit, or modify manually? What decisions did the AI leave to you? ]

Reflection

What Worked Well

[ What did AI-assisted coding do well? Where did it save you time or produce something you wouldn't have built easily on your own? ]

Limitations I Noticed

[ Where did the AI fall short? Did it make mistakes, misunderstand your request, or produce code that didn't work as expected? ]

What This Taught Me About Coding

[ How did vibe coding change how you think about writing code? Does using AI feel like cheating, a tool, or something in between? What is the human's role when AI is doing the heavy lifting? ]

Connections to Digital Culture & Data Analytics
